# Support & Community

Thank you for using Resilient-Networking! We're here to help you succeed.

## 📞 Getting Help

### For Questions & General Support

**GitHub Discussions** (Recommended)
- Feature: Ask questions, share ideas, discuss design
- Response Time: 2-7 days (community dependent)
- Visibility: Helps others with similar questions

**Email Support**
- General Questions: founder@nbr.company
- Response Time: Best effort, 5-10 business days

### For Bug Reports

**GitHub Issues**
- Use Issue template: "Bug Report"
- Include: reproduction steps, expected vs actual, environment details
- Response Time: Critical (24h), High (3 days), Normal (1 week)

### For Security Issues

**Responsible Disclosure**
- Email: founder@nbr.company
- Subject: [SECURITY] <brief description>
- Do NOT open public issues for security vulnerabilities
- We will respond within 24 hours
- See [SECURITY.md](SECURITY.md) for details

### For Feature Requests

**GitHub Discussions or Issues**
- Start a discussion first to gauge interest
- Use Issue template: "Feature Request"
- Include: use case, motivation, proposed solution
- Response Time: 1-2 weeks review

## 🎯 Issue Templates

When opening an issue, use the appropriate template:

### Bug Report
```markdown
## Description
Clear description of the issue.

## Steps to Reproduce
1. Step 1
2. Step 2
3. Step 3

## Expected Behavior
What should happen.

## Actual Behavior
What actually happens.

## Environment
- Python: 3.11.x
- Module: connection-multiplexer
- OS: Linux/macOS/Windows

## Logs
Error message and stack trace.
```

### Feature Request
```markdown
## Description
Clear description of the feature.

## Use Case
How would this be used?

## Proposed Solution
How might this be implemented?

## Alternatives
Other approaches considered?
```

### Question
```markdown
## Question
Your question here.

## Context
Relevant code or configuration.

## What you've tried
Steps you've already taken.
```

## 💡 Tips for Getting Help

### Do
✅ Search existing issues/discussions first
✅ Provide minimal reproducible example
✅ Include error messages and logs
✅ Specify Python version and environment
✅ Be patient and respectful

### Don't
❌ Open duplicate issues
❌ Post on unrelated issues
❌ Use email for non-urgent questions
❌ Share sensitive information publicly
❌ Demand immediate response
